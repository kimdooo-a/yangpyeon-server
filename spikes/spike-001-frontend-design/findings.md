## 2. 현재 디자인 분석

### 강점 (유지)
- **컬러 시스템**: Supabase 스타일 다크 테마가 이미 잘 구축됨 (`#3ECF8E` brand, `#171717~#282828` surface)
- **사이드바**: 모바일 반응형 구현 완료 (햄버거 메뉴 + 오버레이)
- **카드 컴포넌트**: StatCard가 percent bar + MiniChart 지원
- **데이터 폴링**: 3~10초 주기 실시간 갱신

### 약점 (개선 필요)

| 페이지 | 문제점 |
|--------|--------|
| **로그인** | 기능은 충분하나 브랜딩/비주얼 임팩트 부족 |
| **대시보드** | 카드 4개 + 디스크 + 시스템정보 — 정보 계층 평탄 |
| **프로세스** | 테이블만 존재, 요약 카드 없음 |
| **로그** | 터미널 느낌 부족, 줄번호/타임스탬프 구분 없음 |
| **네트워크** | 정보량 적음, 시각적 다이어그램이 텍스트 ASCII |

## 3. 디자인 원칙 (Supabase 참조 + 2026 트렌드)

### 3-1. 색상 계층
```
Level 0 (배경):     #171717  — surface-100
Level 1 (카드):     #1C1C1C  — surface-200
Level 2 (hover):    #232323  — surface-300
Level 3 (active):   #282828  — surface-400
Level 4 (elevated): #2E2E2E  — border

텍스트:
- Primary:   #EDEDED (기존 유지)
- Secondary: #A0A0A0
- Muted:     #6B7280 (gray-500)

상태 색상:
- Success:  #10B981 (emerald-500)
- Warning:  #F59E0B (amber-500)
- Error:    #EF4444 (red-500)
- Info:     #3B82F6 (blue-500)
```

### 3-2. 레이아웃 패턴 (Supabase 참조)
- **PageHeader**: 각 페이지 상단에 제목 + 설명 + 우측 액션
- **PageSection**: 의미 단위로 콘텐츠 그룹화
- **카드 컨테이너**: `bg-surface-200 border border-border rounded-lg`
- **간격**: section 간 `space-y-6`, 내부 `p-5`

### 3-3. 인터랙션
- hover: `hover:bg-surface-300` (밝기 한 단계 올림)
- transition: `transition-colors duration-150`
- 버튼 active: `active:scale-[0.98]` 미세 축소
- focus: `focus:ring-2 focus:ring-brand/50` 포커스 링

## 4. 페이지별 디자인 개선안

### 4-1. 로그인 페이지 (`/login`)

**현재**: 중앙 카드 + 비밀번호 입력 + 버튼
**개선**:

```
┌─────────────────────────────────────────────┐
│                                             │
│            ┌─────────────────┐              │
│            │   🍳             │              │
│            │  양평 부엌       │              │
│            │  서버 대시보드    │              │
│            │                 │              │
│            │ ┌─────────────┐ │              │
│            │ │ 비밀번호     │ │              │
│            │ └─────────────┘ │              │
│            │                 │              │
│            │ [ 로그인 ]      │              │
│            │                 │              │
│            │ ─── v1.0 ───   │              │
│            └─────────────────┘              │
│                                             │
│    미묘한 그라데이션 배경 (brand → dark)      │
│                                             │
└─────────────────────────────────────────────┘
```

- **배경**: 미묘한 radial gradient (brand 색상 5% opacity 중심)
- **로고 영역**: 아이콘 + 텍스트 크기 키움
- **입력 필드**: 더 큰 패딩, 포커스 시 brand 글로우
- **버튼**: brand 그라데이션 + hover 밝기 변화
- **하단**: 버전 정보 또는 "Powered by PM2" 텍스트

**현재**: 테이블만 존재
**개선**:

```
┌────────────────────────────────────────────────────────┐
│ PM2 프로세스                          [모두 재시작]     │

│ └────────┘ └────────┘ └────────┘ └────────┘           │
│                                                        │
│ 검색: [________________]   상태: [전체 ▼]              │
│                                                        │
│ ┌──────────────────────────────────────────────────┐   │
│ │ 이름       상태     CPU   메모리  업타임  액션    │   │

- **요약 카드 추가**: 전체/Online/Stopped/Error 카운트 (클릭 시 필터)
- **검색 + 상태 필터**: 테이블 상단 툴바
- **테이블 개선**: 상태 뱃지 스타일 (dot → 배경색 pill), 액션 아이콘 버튼
- **일괄 액션**: 헤더에 "모두 재시작" 버튼

### 4-4. 로그 뷰어 (`/logs`)

**현재**: 필터 + 모노스페이스 텍스트 영역
**개선**:

```
┌────────────────────────────────────────────────────────┐
│ 로그 뷰어                                              │
│ PM2 프로세스 로그                                       │
├────────────────────────────────────────────────────────┤
│ [전체 ▼] [전체 레벨 ▼] [검색...______] [☑ 자동스크롤] │
│ 표시: 142줄 / 200줄                                    │
├────────────────────────────────────────────────────────┤
│  1 │ 14:32:01 │ INFO  │ dashboard │ Server started...  │
│  2 │ 14:32:02 │ INFO  │ dashboard │ Listening on 3000  │
│  3 │ 14:32:05 │ WARN  │ api       │ Slow query: 2.3s  │
│  4 │ 14:32:08 │ ERROR │ worker    │ Connection refused │
│  5 │ 14:32:10 │ INFO  │ dashboard │ GET /api/system    │
│    │          │       │           │                    │
│    │          │       │           │                    │
│    ▼ (자동 스크롤)                                      │
├────────────────────────────────────────────────────────┤

```

- **줄 번호**: 좌측 고정 열 (gray-600)
- **타임스탬프 열**: 파싱 가능한 로그면 별도 열 표시
- **레벨 뱃지**: ERROR=빨간 pill, WARN=노란 pill, INFO=회색
- **프로세스 열**: 로그 소스 구분 (색상 코딩)
- **하단 상태바**: 현재 위치 / 전체 라인 + 실시간 표시기
- **구조화 로그 파싱**: 가능하면 JSON 로그를 테이블로 변환

**현재**: Tunnel 상태 카드 + ASCII 다이어그램
**개선**:

```

│                                                        │
│ ┌──────────────────┐ ┌──────────────────┐              │
│ │ Tunnel 상태       │ │ 도메인            │              │

│                                                        │
│ ┌─── 네트워크 토폴로지 ─────────────────────────────┐  │
│ │                                                    │  │
│ │  [사용자] ──→ [Cloudflare Edge] ──→ [Tunnel]       │  │
│ │     🌐          ICN (인천)           QUIC          │  │
│ │                                       │            │  │
│ │                               [WSL2 :3000]         │  │
│ │                                  Next.js           │  │
│ │                                   PM2              │  │

│                                                        │
│ ┌─── 연결 정보 ─────────────────────────────────────┐  │
│ │ 프로토콜: QUIC      │ 포트: 3000                   │  │
│ │ 리전: ICN (인천)    │ TLS: Cloudflare Edge         │  │
│ └────────────────────────────────────────────────────┘  │
└────────────────────────────────────────────────────────┘

- **요약 카드 2개**: Tunnel 상태 + 도메인 정보 (분리)
- **시각적 다이어그램**: ASCII → CSS 기반 노드-엣지 다이어그램 (div + flexbox)
- **연결 정보**: 수평 그리드 레이아웃
- **응답 시간** (향후): Tunnel 지연시간 표시 가능성

## 6. 기술적 고려사항

### 추가 의존성 불필요
- SVG 아이콘: 인라인 (외부 라이브러리 없이)
- 다이어그램: CSS flexbox/grid (라이브러리 없이)
- 애니메이션: Tailwind CSS built-in

### 구현 순서 권장
1. 공통 컴포넌트 (PageHeader, StatusBadge, EmptyState)
2. 사이드바 개선
3. 대시보드 (가장 복잡, 핵심 페이지)
4. 프로세스 (요약 카드 + 테이블 개선)
5. 로그 (구조화 + 줄번호)
6. 네트워크 (다이어그램)
7. 로그인 (마지막, 독립적)

### 성능 영향
- 추가 API 호출: 프로세스 요약 카드 (기존 /api/pm2 응답에서 계산 가능)
- 번들 크기: 외부 라이브러리 추가 없음, 영향 최소
