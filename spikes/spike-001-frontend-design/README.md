# Spike-001: 프론트엔드 디자인

## 요약

| 항목 | 값 |
|------|-----|
| 주제 | 각 웹페이지 프론트엔드 디자인 개선 |
| 난이도 | Level 3 (풀 스파이크) |
| 시작일 | 2026-04-06 |
| 상태 | 완료 |
| 판정 | **Go** |

## 배경

양평 부엌 서버 대시보드의 5개 페이지(로그인, 대시보드, 프로세스, 로그, 네트워크)에 대한
디자인 품질 개선을 위해 레퍼런스 조사 및 페이지별 개선안을 수립.

## 리서치 범위

1. Supabase Design System (Layout, Tables 패턴)
2. 2026 Dashboard UI 트렌드 (Muzli, AdminLTE, Dark Mode 가이드)
3. 실시간 로그 뷰어 UI 패턴 (Logdy, PatternFly)
4. 다크 모드 디자인 모범 사례

## 핵심 발견

### 유지할 것
- 기존 Supabase 스타일 다크 테마 컬러 시스템 (`#3ECF8E` brand)
- surface 4단계 계층 (`#171717` ~ `#282828`)
- 카드 기반 레이아웃 구조
- 모바일 반응형 사이드바

### 개선할 것
- **정보 계층 강화**: 요약 카드 → 상세 테이블 2단계 구조 (프로세스, 대시보드)
- **시각적 일관성**: PageHeader 표준화, 로딩/빈 상태 패턴 통일
- **로그 뷰어**: 줄번호, 레벨 뱃지, 프로세스 컬러 코딩
- **네트워크**: ASCII → CSS 기반 토폴로지 다이어그램
- **사이드바**: 이모지 → SVG 아이콘, 활성 표시 강화
- **로그인**: 브랜딩 강화 (그라데이션 배경, 포커스 글로우)

### 기술적 결정
- **추가 라이브러리 없음**: 모든 개선을 Tailwind CSS + 인라인 SVG로 구현
- **기존 API 유지**: 새 API 엔드포인트 불필요 (기존 응답 데이터 활용)
- **점진적 개선**: 공통 컴포넌트 → 페이지 순서대로
