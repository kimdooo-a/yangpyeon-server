## 토픽 4: C3 — API 레이어 (`d967746`)

세 라우트 작업:
1. `schema/route.ts`에 `primaryKey`/`compositePk` 응답 필드 추가
2. `[table]/route.ts`에 `POST` 핸들러 append (INSERT)
3. `[table]/[pk]/route.ts` 신규 생성 (PATCH/DELETE)

주요 발견:
- plan의 `import { writeAuditLog as writeAuditLogDb } from "@/lib/audit-log-db"` 앨리어싱은 실 export 이름(`writeAuditLogDb`)과 불일치 — 직접 import로 수정

- plan은 PATCH/DELETE에 function declaration 중첩 withRole 패턴을 사용했으나, 기존 Phase 14a 관습(`export const HANDLER = withRole(...)`)과 일치하도록 단순화
- dev 서버 curl 통합 테스트는 `/api/auth/login-v2` CSRF 토큰 요구로 막힘 — C5 프로덕션 E2E로 위임

**결론**: tsc 통과. 통합 테스트는 Task 12 프로덕션 경로에서 수행.
