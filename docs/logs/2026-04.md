- **elkjs 비동기 레이아웃 패턴**: `elk.layout()`은 Promise. 초기 마운트 시 position(0,0)로 모든 노드 배치 후 Promise resolve → `setNodes(positioned)` 재주입. 노드 높이를 컬럼수 비례(22px/row + 44px header + 8px footer)로 ELK에 전달 → FK 엣지 겹침 방지.
- **localhost:3000 정체**: `Get-NetTCPConnection -LocalPort 3000` → `wslrelay.exe` 소유 → dev 서버가 아닌 WSL2 PM2 프로덕션. 코드 변경은 빌드+배포 전까지 무효 확인. 세션 17 초반 404 증상의 원인.
- **Windows 빌드 블로킹**: `lightningcss-win32-x64-msvc` optional native binary 미설치로 `next build` 실패. WSL2 Linux에는 올바른 바이너리 존재하므로 실제 배포는 정상. `/ypserver` 스킬 Phase 1 항상 실패하는 환경 이슈로 승격 → 스킬 보강 과제화.
- **기본 SQL 오류**: 세션 14/15 스캐폴드 시 `"User"` 하드코딩. 실제 Prisma 마이그레이션은 `@@map("users")`로 소문자 `users` 생성. Monaco→state→API 파이프라인 정상 검증 도중 드러남. 한 라인 수정 + 재배포로 해결.
