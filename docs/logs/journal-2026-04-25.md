### [1] standalone 로그 확인 → invalid ELF header 진단

- **사용자**: "standalone 로그 확인해봐... 로그인이 안돼."
- **논의**: WSL PM2 경유로 `~/ypserver/logs/ypserver-err.log` 확인. `ERR_DLOPEN_FAILED` + `invalid ELF header`가 2026-04-19 22:21부터 04-21 22:02까지 6건 발견. `file(1)`로 `.next/node_modules/better-sqlite3-*/build/Release/better_sqlite3.node` 검증 → `PE32+ executable (DLL) for MS Windows` 확정. Windows에서 빌드된 DLL이 Linux dlopen에 투입된 고전적 크로스 플랫폼 문제. `node_modules/better-sqlite3/build/Release/`의 Linux ELF 바이너리는 `install-native-linux.sh`의 `npm rebuild`로 정상이나 NFT 해시 디렉토리는 손대지 않음을 확인.
- **결론**: 임시 수정(ELF `.node`를 NFT 해시 디렉토리에 덮어쓰기) + PM2 재시작으로 복구. curl `POST /api/auth/login` → 403 CSRF (핸들러 정상 도달).
- **파일**: (진단만, 수정 없음)
