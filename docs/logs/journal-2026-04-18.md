## 토픽 7: 2차 근본 버그 — raw SQL UPDATE auto-bump 부재

스크립트 수정 후 재실행 → **C2 CONFLICT 여전히 FAIL** (200 반환). 분석:
- C1 (expected=T0, 현재=T0) → UPDATE 성공 → 그런데 반환된 updated_at도 T0 (변하지 않음!)
- C2 (expected=T0, 현재=T0) → 여전히 T0==T0 → 잠금 통과 → 200

**근본 원인**: Prisma `@updatedAt`는 클라이언트 레벨 동작. raw `UPDATE folders SET name=$1 WHERE …`는 `updated_at` 컬럼을 건드리지 않음. DB `DEFAULT now()`는 INSERT에만 발동.

**해결**: API가 `hasUpdatedAtCol && !userSetUpdatedAt`이면 `SET ..., updated_at = NOW()`를 자동 주입. Prisma `@updatedAt` 동등 동작을 raw SQL 레이어에서 재현.

commit `00d4e79`: API auto-bump + 스크립트 seed 수정 + tsconfig scripts 제외.

## 토픽 15: E2E 2차 FAIL — TIMESTAMP 정밀도

재배포 후 B1(정상 PATCH 락 일치) 여전히 409 CONFLICT. INITIAL_UPDATED_AT과 현재 updated_at이 동일한 ISO 문자열로 보이는데도 WHERE 매칭 실패.

**근본 원인**: 테스트 테이블 `TIMESTAMP` (µs 정밀도) vs Prisma 실제 `TIMESTAMP(3)` (ms). pg 드라이버 ISO 직렬화는 ms 절단 → `Date(ms)` 재파라미터 시 DB에 저장된 µs와 mismatch. `TIMESTAMP(3)`로 재생성 → E2E 3차 B1~B9 전 PASS.
