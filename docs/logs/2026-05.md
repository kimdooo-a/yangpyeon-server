**주요 논의**:

- **종료 순서 원칙**: 외부 트래픽 입구인 `cloudflared` (PM2 id 1) 를 먼저 끊고 `ypserver` (id 2) 정지 — 그래야 종료 도중 502 안 뜸. 재기동은 역순 (`ypserver → cloudflared`). graceful shutdown 은 `ecosystem.config.cjs` 의 `kill_timeout: 8000` 으로 8초 동안 in-flight 요청 처리 후 SIGKILL.
- **PM2 PATH 함정**: `wsl -d Ubuntu -- bash -lc 'pm2 list'` 는 nvm PATH 미로드로 `command not found`. 반드시 `bash -ilc` (interactive login) 사용 — 매뉴얼 트러블슈팅 섹션에 명시.
- **WSL 한글 변수 함정**: `wsl ... bash -ilc 'A="$X"; pandoc "$A"'` 형태에서 한글/특수문자 변수가 외부 셸 quoting 단계에서 빈 문자열로 잘림. 인라인 경로 직접 사용 또는 `/tmp` 경유 + `cp` 로 회피.
- **v1 양식 분석 5차 반복** — 결정적 발견: v1 docx 는 사용자가 손으로 꾸민 문서로, **양식이 styles.xml 이 아니라 document.xml 의 단락별 인라인 오버라이드** 에 있었음. pandoc 은 인라인 오버라이드를 만들지 못하므로, 같은 비주얼을 styles.xml 에 baked-in 해야 적용됨. 이 패턴은 향후 모든 docx 템플릿 작업에서 재사용 가치 높음 (CK 후보).
- **mammoth+Playwright 시각 비교의 한계**: mammoth 가 SourceCode 단락을 `<pre>` 로 변환 안 함, 단락 spacing 미반영. true 시각 검증은 LibreOffice/Word 필요. sudo 패스워드 없이 LibreOffice 설치는 불가 — 사용자 측 ! 명령으로 안내.
- **파일박스 4단 게이트 진단**: ① Cloudflare Tunnel 무료/Pro = request body 100MB (1.4GB 14× 초과, 첫 차단) ② `MAX_FILE_SIZE=50MB` (28× 초과) ③ `DEFAULT_STORAGE_LIMIT=500MB` (일반 유저 2.8× 초과) ④ `request.formData()` 전체 메모리 로딩 + PM2 max_memory 512MB → OOM. 브라우저 progress 무한 루프 = Cloudflare RST → Chrome 자동 재시도 → 0%부터 다시.
