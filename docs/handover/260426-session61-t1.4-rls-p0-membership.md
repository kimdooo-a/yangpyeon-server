### 토픽 5: P0-membership Inline 구현

> T1.4 worktree 정리 후 (PowerShell long-path remove) inline 진행. 별도 agent dispatch 오버헤드 회피 — ~30분 작업.

기존 `src/lib/tenant-router/membership.ts` 검토: stub (항상 null), TenantMembershipRow 인터페이스 = `{ role: TenantRole }`, 사용 위치는 api-guard-tenant.ts:157 + api-guard-tenant.test.ts (mock).

핵심 결정: **TenantMembership = Tenant-bypass 카테고리**.
- 멤버십 조회는 tenant context가 *결정되기 전* 단계 (사용자가 어느 tenant에 속하는지 본 query 가 판정 중) → RLS USING 절이 `app.tenant_id` 를 요구하면 self-defeating.
- (tenantId, userId) 양쪽 명시 bind parameter로 cross-tenant 안전.
- 따라서 base prisma 직접 사용 + ESLint inline-disable + 사유 코멘트.

prisma 모델 추가:
```prisma
model TenantMembership {
  id        String   @id @default(uuid()) @db.Uuid
  tenantId  String   @map("tenant_id") @db.Uuid
  userId    String   @map("user_id")
  role      String   @default("MEMBER")
  createdAt DateTime @default(now()) @map("created_at") @db.Timestamptz(3)
  updatedAt DateTime @default(now()) @updatedAt @map("updated_at") @db.Timestamptz(3)

  tenant Tenant @relation("TenantMemberships", fields: [tenantId], references: [id], onDelete: Cascade)
  user   User   @relation("UserTenantMemberships", fields: [userId], references: [id], onDelete: Cascade)

  @@unique([tenantId, userId], map: "tenant_memberships_tenant_id_user_id_key")
  @@index([userId])
  @@map("tenant_memberships")
}
```

Migration `20260427120000_p0_tenant_membership/migration.sql`:
- CREATE TABLE + composite unique + user_id index
- RLS 명시적 미적용 (Tenant-bypass)
- 'default' tenant 활성 사용자 OWNER 시드 ON CONFLICT DO NOTHING (멱등)
- migration_rollback.sql sibling

membership.ts 본문 교체:
- stub `return null` → `(prisma as any).tenantMembership.findUnique({ where: { tenantId_userId: ... }, select: { role: true } })`
- role narrow: TenantRole literal 외 값은 fail-closed null 반환 (stale row 방어)
- @ts-nocheck 영향으로 any 캐스트 정당화 코멘트
- eslint-disable + 사유 코멘트 (no-restricted-imports — 인증 인프라 base prisma 직접 사용 정당)

**결론**: P0-membership commit `6af4154`. 5/5 게이트 PASS. 통합 부채 1번 마감.
