# 인수인계서 (Handover)

> AI 세션 간 컨텍스트 유지를 위한 프로토콜

---

## 세션 시작 프로토콜

> 글로벌 규칙 기준: `current.md` + 최신 handover + `next-dev-prompt.md` 확인

```
1. status/current.md 확인
   - 현재 Phase 확인
   - 진행 중인 작업 파악
   - 세션 요약표 확인

2. 최신 handover 파일 확인
   - 이전 세션 작업 내용
   - 남은 작업 파악

3. next-dev-prompt.md 확인 (있을 경우)
   - 추천 다음 작업
   - 알려진 이슈
```

필요 시 추가 확인:
- 프로젝트 구조 파악 (주요 디렉토리, 핵심 파일)
- 규칙 파일 확인 (`docs/rules/_index.md`)
- 이번 세션 목표 명확화 및 우선순위 결정

---

## 세션 종료 프로토콜 (4단계)

```
1단계: status/current.md 세션 요약표에 1행 추가
   - 세션 번호, 날짜, 제목, 아카이브, 인수인계서 링크

2단계: logs/ 아카이브에 상세 기록
   - 해당 기간의 아카이브 파일에 추가
   - 구현한 기능, 수정한 파일, 이슈 등

3단계: handover/ 인수인계서 작성
   - 아래 템플릿 사용
   - 관리 전략에 따라 정리

4단계: next-dev-prompt.md 갱신
   - 최근 완료 작업, Git 상태, 추천 다음 작업
```

---

## 파일명 규칙

프로젝트 관례에 따라 택일:
- **형식A**: `YYYYMMDD_HHMM_프로젝트명_handover.md` (예: `20250116_1430_monet_handover.md`)
- **형식B**: `YYYY-MM-DD-sessionN.md` (예: `2026-02-08-session34.md`)

---

## 인수인계 템플릿

```markdown
# 인수인계서
> 작성: YYYY-MM-DD HH:MM
> 프로젝트: [프로젝트명]
> 현재 Phase: [Phase X]

## 오늘 완료한 작업
-

## 진행 중인 작업
- 현재 상태:
- 다음 단계:

## 다음에 해야 할 작업
1.
2.

## 정제 상태
| 단계 | 상태 | 미완료 항목 |
|------|------|------------|
| 1차 (구조) | ✅/⚠️/❌ | - |
| 2차 (품질) | ✅/⚠️/❌ | - |
| 최종 (보안) | ✅/⚠️/❌ | - |

## 주의사항 / 이슈
-

## 관련 파일
-
```

---

## 정제 상태 기호

| 기호 | 의미 |
|------|------|
| ✅ | 완료 |
| ⚠️ | 진행중 (미완료 항목 있음) |
| ❌ | 미시작 또는 차단 이슈 있음 |

---

## 관리 전략

프로젝트 특성에 따라 택일:

| 전략 | 규칙 | 적합한 경우 |
|------|------|-------------|
| **max 5 순환** | 최대 5개 유지, 초과 시 archive/로 이동 | 단기/소규모 |
| **영구 보존** | 모든 인수인계서 보존, `_index.md`로 관리 | 장기/다수 참여 |
