## 문제

ADR-032 (Cloudflare R2 hybrid) 가 세션 71 에서 PROPOSED → ACCEPTED 동일 세션 승격되었고, "결정자: 프로젝트 오너 — R2 토큰 발급 + PoC 6/6 합격 후 V1 옵션 A 즉시 적용" 라고 명시되어 있었다. 즉 **운영자 본인이 sign-off 한 결정**.

그러나 세션 77 진행 중 운영자가 R2 도입 자체에 의문 제기:

> "난 돈을 안쓰려고 내 집에서 내 컴퓨터로 서버를 만든것인데 왜 또 지출이 필요한 서비스를 사용하게 하는거야??"

운영자 본인 sign-off 가 있었음에도 6일 후(s71→s77) 가치관 충돌이 표면화. **결정자 라인의 sign-off 가 가치관 충돌 점검을 보장하지 않음**.

증상:
- ADR 본문 §"비용 모델" 에 "$0.06/월 ... 현실적으로 무료 (운영자 단독 사용 컨텍스트)" 명시
- §3 옵션 매트릭스의 옵션 E (SeaweedFS) 거부 이유 = "운영 부담 큼"
- 그러나 운영자 가치관 = "운영 부담 감수해도 자가호스팅 선호"
- ADR 결정 프로세스 어디에도 "운영자 가치관 정합성" 점검 항목 없음

## 원인

ADR 결정 프로세스가 다음을 가정:
1. 운영자 sign-off = 가치관 충돌 점검 완료
2. PoC 합격 = 운영 가능성 검증 완료
3. 비용 모델 우위 = 결정 정당성 확보

**누락된 가정**:
- 운영자가 결정 시점에 **"이 결정이 내 핵심 가치관과 정합하는가"** 를 명시적으로 점검했는지
- "비용 ≈ 0" 의 해석이 운영자 입장에서 "외부 서비스 0원" vs "전기료 외 진정한 0원" 중 어느 쪽인지
- "운영 부담" 의 평가가 운영자 가치관 (자가호스팅 선호) 과 정렬되는지

ADR 템플릿 (ADR-022~032) 의 §"검토한 대안 및 거부 이유" 가 **운영 효율성 / 비용 / 기술적 위험** 만 평가하고 **운영자 가치관 정합성** 을 별도 평가하지 않음.

## 해결

### 즉시 (세션 77)

옵션 C 채택 — R2 폐기 + SeaweedFS 자가호스팅. ADR-032 → ADR-033 supersede 경로. 새 터미널용 7 PHASE 마이그레이션 풀 패키지 프롬프트 작성 (저널 [5]).

### 향후 ADR 프로세스 보강 (다음 ADR-034 부터 적용 후보)

ADR 템플릿 §"검토한 대안" 다음에 §"운영자 가치관 정합성 점검" 신규 섹션:

```markdown
## N. 운영자 가치관 정합성 점검

다음 6개 항목 중 충돌 가능성이 1건이라도 있으면 결정 보류 + 가치관 명시 후 재논의:

1. **자가호스팅 선호** — 외부 SaaS/API 의존이 가치관과 충돌하는가?
2. **비용 0 절대 가치** — "무료 티어 안에서 0원" vs "전기료 외 진정한 0원" 어느 쪽인가?
3. **외부 의존 회피** — 외부 서비스 정책 변경 위험을 감수할 수 있는가?
4. **운영 부담 감수 의지** — 자가호스팅의 운영 부담이 가치관에 부합하는가?
5. **데이터 통제** — 외부에 데이터를 두는 것에 대한 가치관은?
6. **장기 vs 단기** — 단기 편의성과 장기 자율성 중 우선순위는?

각 항목에 운영자 본인이 명시적으로 답변. 구두 동의 + sign-off 만으로는 불충분.
```

### 본 패턴의 일반화

**모든 외부 서비스 도입 ADR (R2 / B2 / Sentry / Resend / Cloudflare * 등)** 결정 시 위 6 항목 점검 강제. 1인 운영자 컨텍스트의 BaaS 프로젝트는 운영자 가치관이 결정의 절대 기준.

## 교훈

1. **결정자 sign-off ≠ 가치관 정합성 보장**. 운영자가 토큰을 발급했어도 가치관 충돌 점검을 명시적으로 한 건 아님 — "PoC 합격 후 즉시 적용" 의 페이스가 가치관 점검 단계를 건너뛸 수 있음.
2. **ADR 비용 모델 표현에 주의**. "$0.06/월" 같은 absolute 숫자는 객관적이지만, **"운영자 가치관 기준 의미"** (예: "자가호스팅 외 어떤 비용도 가치관에 반함") 를 명시하지 않으면 sign-off 시 누락 가능.
3. **6일 안의 supersede 는 결정 프로세스의 갭을 알려주는 신호**. 즉시 회수 후 ADR 템플릿 보강.
