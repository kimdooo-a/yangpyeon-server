# ADR-001: 프론트엔드 디자인 개선 접근법

## 상태
승인 (2026-04-06)

## 컨텍스트
양평 부엌 서버 대시보드 5개 페이지의 디자인 품질을 개선해야 한다.
현재 기능은 완성되어 있으나 정보 계층, 시각적 일관성, UX 품질을 높일 여지가 있다.

## 검토한 옵션

### 옵션 A: UI 라이브러리 도입 (shadcn/ui, Radix)
- **장점**: 풍부한 컴포넌트, 접근성 내장
- **단점**: 의존성 추가, 기존 스타일 충돌 가능, 번들 증가
- **판정**: 기각 — 현재 규모(5페이지)에 과도

### 옵션 B: Tailwind CSS + 인라인 SVG로 자체 개선 (선택)
- **장점**: 의존성 없음, 기존 스타일 시스템 유지, 번들 영향 최소
- **단점**: 복잡한 컴포넌트(모달, 드롭다운) 직접 구현 필요
- **판정**: **채택** — 현재 프로젝트 규모에 최적

### 옵션 C: 디자인 시스템 라이브러리 구축
- **장점**: 재사용성 극대화
- **단점**: 과도한 추상화, 단일 프로젝트에 불필요
- **판정**: 기각 — YAGNI

## 결정
**옵션 B** 채택: 외부 라이브러리 없이 Tailwind CSS + 인라인 SVG로 전체 디자인 개선.

## 근거
1. 프로젝트 규모(5페이지)에 UI 라이브러리 도입은 과도
2. 기존 Supabase 스타일 컬러 시스템이 잘 구축되어 있음
3. 번들 크기 증가 없이 시각적 품질 개선 가능
4. PM2 기반 배포 환경에서 가벼운 번들 유지가 중요

## 영향
- 공통 컴포넌트 3~4개 추가 (PageHeader, StatusBadge, EmptyState, Icon)
- 기존 API 변경 없음
- 예상 작업량: 7단계 점진적 구현
