## 1. TCO 정의

### 1.1 계산 기간 및 범위

**기간**: 2026-04 ~ 2029-04 (3년, 36개월)

**TCO 구성 요소**:

| 요소 | Supabase Cloud | 양평 자체호스팅 |
|-----|--------------|-------------|
| **인프라 SaaS** | Supabase Pro/Team 구독료 | $0 (WSL2 + 기존 PC) |
| **외부 서비스** | Supabase 내장 (Auth, Storage, Edge) | Cloudflare Tunnel, B2, 가비아 |
| **AI/LLM** | 외부 AI API 별도 계약 | Anthropic BYOK ($0~5/월) |
| **도구 및 CI** | GitHub Actions (외부) | GitHub Actions (무료) |
| **인력 시간가치** | 운영 유지보수만 | 구축 870h + 운영 유지보수 |
| **도메인** | 별도 | 가비아 연 ~$18 |

**포함하지 않는 항목**:
- Windows 11 Pro 라이선스 (기존 구매)
- 인터넷 회선 비용 (KT 가정용, 기존 계약)
- 물리 하드웨어 감가상각 (기존 PC 활용, 추가 구매 없음)
- 전기요금 (미미한 수준, 추정 $5~10/년)

### 1.2 비용 단위 및 환율

- 기준 통화: USD
- 가비아 도메인: 연 $18 (약 24,000원 기준, KRW/USD 1,350 적용)

- 순 ROI(표준): ($950 - $250) / $250 = **280%**
- 순 ROI(성장): ($2,150 - $250) / $250 = **760%**

## 6. 인력 시간가치 회계

### 6.1 870h 구축 공수의 시간가치 환산

Wave 4 청사진에서 산정된 Phase 15-22 총 공수: **870h** (README.md §Wave 4 핵심 발견 5번)

**시나리오 1: 시간가치 계산에서 제외 (오너 자신의 학습/경험으로 간주)**

| 항목 | 값 |
|------|-----|
| 구축 공수 | 870h |
| 시간가치 적용 | 없음 (본인 취미/학습 투자) |
| 3년 절감 | $950 ~ $2,150 |
| 결론 | **ROI 280~760%, 3년 후 $950~2,150 순 절감** |

**시나리오 2: 프리랜서 시장가 기준 시간가치 포함**

한국 시니어 풀스택 개발자 프리랜서 평균 시급 기준:
- 국내 기준: 시급 8만원 (약 $60/h, KRW/USD 1,350 기준)
- 해외 주요 시장(Upwork 등) 시니어 풀스택 평균: $50~80/h → 중간값 $60/h 적용

| 항목 | 값 |
|------|-----|
| 구축 공수 | 870h |
| 프리랜서 시급 | $60/h |
| **870h × $60 = $52,200** | 시간가치 |
| 3년 절감 (표준) | $950 |
| 3년 절감 (성장) | $2,150 |
| **시간가치 포함 순 손익 (표준)** | $950 - $52,200 = **-$51,250** |
| **시간가치 포함 순 손익 (성장)** | $2,150 - $52,200 = **-$50,050** |

**해석**: 시간가치를 프리랜서 외주 단가로 환산하면 자체호스팅이 경제적으로 불리하다. 그러나 이 프로젝트의 목적은 **비용 절감이 아니라 기술 역량 확보 + 데이터 주권 + Supabase 100점 동등 아키텍처 연구**에 있다.

### 6.2 인력 시간가치 포함 ROI 해석 기준

| 목적 | 적용 시나리오 | ROI |
|-----|------------|-----|
| 순수 비용 절감 | 시나리오 1 | 280~760% |
| 외주 대체 비용 관점 | 시나리오 2 | 음수 (학습 투자) |
| 기술 역량 + 데이터 주권 | 비재무 ROI | 측정 불가 (전략적 가치) |

**프로젝트 공식 입장**: ROI는 **시나리오 1** 기준으로 계산 (본인 투자). 870h는 비용이 아닌 기술 자산 취득으로 분류.

## 7. 비용 변동 시나리오

### 7.1 시나리오 A: B2 스토리지 100GB 초과

**발생 조건**: SeaweedFS 로컬 티어 한계 도달 + B2 오프로드 급증

| 항목 | 정상 (50GB) | 초과 (100GB) | 초과 (500GB) |
|------|-----------|-----------|-----------|
| B2 스토리지 비용/월 | $0.30 | $0.60 | $3.00 |
| B2 다운로드 비용/월 | $0.10 | $0.20 | $1.00 |
| 합계/월 | $0.40 | $0.80 | $4.00 |
| 3년 합계 | $14 | $29 | $144 |
| 전체 TCO 변동 | $250 → $278 | $250 → $293 | $250 → $380 |

## 9. Supabase로 회귀 시 마이그레이션 비용

### 9.1 회귀 시나리오 정의

자체호스팅이 실패하거나 운영 부담이 임계점을 초과할 경우 Supabase Cloud로 복귀하는 백업 시나리오.

**회귀 트리거 조건**:
1. 1인 운영자 장기 사고 (BCP-3 발동, §11 참조) — R-10 연계
2. WSL2/하드웨어 장애로 복구 불가 — R-20 연계
3. 전체 TCO가 Supabase Cloud 표준을 초과하는 상황 지속 6개월+
4. 보안 침해로 전체 재구축 필요

### 9.2 마이그레이션 비용 추정

#### 데이터 마이그레이션 비용

| 항목 | 예상 시간 | 예상 비용 |
|------|--------|--------|
| PostgreSQL → Supabase DB | 8h | 데이터 전환 시간 |
| SeaweedFS + B2 → Supabase Storage | 4h | 대역폭 비용 $0.10/GB × 100GB = $10 |
| Auth 세션/사용자 마이그레이션 | 4h | — |
| API 레이어 재작성 (Next.js Supabase SDK) | 20-40h | — |
| 환경 변수/시크릿 이전 | 2h | — |
| **합계** | **38~58h** | **$10~30 직접 비용** |

#### 마이그레이션 인력 비용 (시간가치 기준)

| 기준 | 시간 | 비용 |
|------|------|------|
| 자체 수행 (프리랜서 단가 $60/h) | 38~58h | $2,280~$3,480 |
| 최소 수행 (도구 활용) | 20h | $1,200 |

#### Supabase Cloud 초기 셋업 비용

- Supabase 계정 설정: 1h
- 환경 변수 구성: 2h
- DNS 전환 (가비아 → Cloudflare → Supabase): 2h
- 테스트 및 검증: 8h
- **셋업 합계**: 13h → $780

#### 회귀 총 비용

| 항목 | 최소 | 최대 |
|------|------|------|
| 직접 비용 (데이터 전환 대역폭) | $10 | $30 |
| 마이그레이션 인력 (시간가치) | $1,200 | $3,480 |
| Supabase 셋업 시간가치 | $780 | $780 |
| 서비스 중단 손실 (추정) | $0 | $100 |
| **회귀 총 비용** | **$1,990** | **$4,390** |

**결론**: Supabase로 회귀 시 $2,000~$4,400의 추가 비용 발생. 이는 자체호스팅 유지의 강력한 이유가 된다.

### 9.3 회귀 방지 설계 원칙

자체호스팅을 포기하지 않도록 회귀 비용을 높이는 설계 원칙:

1. **데이터 표준 준수**: PostgreSQL 표준 스키마 유지 → Supabase가 그대로 수용 가능
2. **API 호환성**: REST API 응답 형식을 Supabase SDK 형식과 유사하게 설계 (ADR-012)
3. **환경 변수 추상화**: 모든 외부 서비스 참조를 환경 변수로 격리 → 교체 용이
