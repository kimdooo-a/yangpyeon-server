# 이미지 파일 관리 규칙

## 저장 위치
```
public/
└── images/
    ├── icons/        → 아이콘, 파비콘
    ├── og/           → OG 이미지 (소셜 공유용)
    ├── hero/         → 히어로 섹션 배경
    ├── ui/           → UI 요소 (empty state, 로딩 등)
    └── content/      → 콘텐츠용 이미지
```

## 파일명 규칙
- 소문자, 하이픈(-) 사용
- 용도-설명-크기.확장자
- 예시:
  - `icon-logo-512.png`
  - `og-default-1200x630.jpg`
  - `hero-main-1920x1080.jpg`
  - `ui-empty-cart-400x300.png`

## 필수 이미지 목록
| 용도 | 크기 | 위치 | 파일명 예시 |
|-----|------|------|------------|
| 파비콘 | 512x512 | icons/ | icon-favicon-512.png |
| OG 기본 | 1200x630 | og/ | og-default-1200x630.jpg |
| 히어로 배경 | 1920x1080 | hero/ | hero-main-1920x1080.jpg |

## 이미지 생성 후 체크리스트
Antigravity에서 이미지 생성 후:
1. [ ] 적절한 폴더에 저장
2. [ ] 파일명 규칙 준수
3. [ ] 크기 최적화 (필요시 압축)
4. [ ] 코드에서 경로 연결

## 최적화 규칙
- JPG: 사진, 그라데이션 배경 (품질 80-85%)
- PNG: 투명 배경, 아이콘, UI 요소
- SVG: 단순 아이콘, 로고 (가능한 경우)
- WebP: 웹 최적화 필요 시 (JPG/PNG 대체)

## 크기 가이드
| 용도 | 권장 크기 | 최대 용량 |
|-----|----------|----------|
| 파비콘 | 512x512 | 50KB |
| OG 이미지 | 1200x630 | 200KB |
| 히어로 배경 | 1920x1080 | 300KB |
| UI 일러스트 | 400x300 | 100KB |
| 썸네일 | 600x400 | 100KB |

## 이미지 요청 → 저장 흐름
```
1. Claude Code가 이미지 프롬프트 생성
   ↓
2. 사용자가 Antigravity에서 이미지 생성
   ↓
3. 다운로드 후 적절한 폴더에 저장
   - public/images/[카테고리]/[파일명]
   ↓
4. Claude Code에게 "이미지 저장했어" 알림
   ↓
5. Claude Code가 코드에 이미지 경로 연결
```

## 금지 사항
- public/images/ 외 위치에 이미지 저장
- 한글 파일명 사용
- 공백, 특수문자 사용
- 원본 고용량 이미지 그대로 사용
