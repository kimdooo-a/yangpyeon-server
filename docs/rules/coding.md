# 코딩 규칙 (범용)

## 범용 원칙

- **기존 패턴 우선**: 새 패턴 도입 전 기존 코드 패턴 먼저 확인
- **순환 의존성 금지**: A → B → A 의존 구조 절대 금지
- **에러 핸들링 필수**: 외부 호출(API, DB, 파일)에 에러 처리
- **async/await 사용**: 콜백 체이닝, then 체이닝 지양
- **콘솔 로그 커밋 금지**: 디버깅용 로그는 커밋 전 제거
- **하드코딩 금지**: URL, 키, 포트 등은 환경변수 또는 설정 파일 사용

## 파일 구조

### 소규모 프로젝트 (플랫 구조)
```
src/
├── controllers/ 또는 components/
├── services/ 또는 hooks/
├── models/ 또는 types/
└── utils/ 또는 lib/
```

### 중대규모 프로젝트 (모듈/도메인 기반)
```
src/
├── [도메인A]/
│   ├── controller 또는 components/
│   ├── service 또는 hooks/
│   └── model 또는 types/
├── [도메인B]/
│   └── ...
└── shared/ 또는 global/
```

적용 기준: 도메인/기능 모듈이 3개 이상이거나 팀 협업 시

## 리팩토링 판단

아래 중 2개 이상 해당 시 리팩토링 고려:
- 파일 1000줄 이상
- 3개 이상 관심사 혼합
- import 15개 초과
- 테스트 작성 어려움 (모킹 과다)

리팩토링 원칙:
- 기존 동작 100% 유지 (구조 변경만)
- 단방향 의존성 (역방향 금지)
- 점진적 적용 (가장 큰 파일부터)
