# 페이지 네비게이션 연결성 규칙

> 상위: [CLAUDE.md](../../CLAUDE.md) → [rules/_index.md](./_index.md) → **여기**
> 버전: 1.1 | 최종 업데이트: 2026-02-16

> **kdyweb 스킬 사용 프로젝트:** `/kdyweb` 스킬의 `page` 모드가 이 규칙의 모든 항목을 포함하는 상위 호환 워크플로우입니다. kdyweb 사용 시 이 문서는 참고용이며, `/kdyweb --page` 모드의 7단계 워크플로우를 따르세요.

---

## 핵심 원칙: "페이지는 섬이 아니다"

**모든 페이지는 루트(`/`)에서 클릭만으로 도달 가능해야 한다.**

URL 직접 입력으로만 접근 가능한 페이지는 죽은 페이지다.
라우터에만 등록되고 어디서도 링크되지 않은 페이지는 존재하지 않는 것과 같다.

```
✅ 올바른 연결 (풀뿌리 구조)

        ┌─ 로그인 ←→ 비밀번호찾기
        │     ↓
홈(/) ──┼─ 회원가입
        │     ↓
        └→ 대시보드 ──┬─ 목록 → 상세 → 편집
                      ├─ 프로필 ←→ 설정
                      └─ 알림

✅ 모든 노드가 연결됨 (고아 없음)
```

```
❌ 잘못된 구조 (죽은 페이지)

홈(/) → 대시보드 → 프로젝트

        /admin        ← 어디서도 링크 안 됨 💀
        /reports      ← 어디서도 링크 안 됨 💀
        /invite       ← 어디서도 링크 안 됨 💀
```

## 페이지 생성 시 필수 4단계

새 페이지를 만들 때 **반드시 아래 4단계를 모두 수행**한다.
1단계만 하고 끝내는 것은 미완성이다.

### 1단계: 페이지 파일 생성
- 컴포넌트/라우트 파일 작성

### 2단계: 라우터 등록
- 라우팅 설정에 경로 추가
- 프레임워크별 방식 준수 (Next.js app router, React Router 등)

### 3단계: 진입점 연결 (가장 중요)
아래 중 **최소 1개 이상** 반드시 수행:

| 연결 방식 | 예시 |
|-----------|------|
| GNB/Header에 링크 추가 | 메인 메뉴 항목으로 추가 |
| 사이드바 메뉴에 추가 | 관리자/대시보드 사이드 네비게이션 |
| 부모 페이지에서 버튼/링크 | 목록 → 상세, 대시보드 → 설정 |
| 관련 페이지에서 상호 링크 | 로그인 ↔ 회원가입, 프로필 ↔ 설정 |
| Footer 링크 | 약관, 개인정보처리방침 등 |

### 4단계: 네비게이션 맵 업데이트
- `docs/references/_NAVIGATION_MAP.md` 라우트 테이블에 행 추가
- 진입점과 이동 대상을 명시
- **kdyweb 사용 프로젝트:** `_NAVIGATION_MAP.md` 대신 `docs/references/_WEB_CONTRACT.md`의 라우트 레지스트리를 갱신

## 연결 체크리스트

새 페이지 생성 시 아래 4개 질문에 반드시 답할 것:

```
□ 이 페이지에 어떻게 들어오는가? (진입점)
□ 이 페이지에서 어디로 갈 수 있는가? (이동 대상)
□ 작업 완료 후 어디로 돌아가는가? (복귀 경로)
□ GNB/사이드바에 표시할 필요가 있는가? (네비게이션 노출)
```

**하나라도 답할 수 없으면 페이지 생성이 완료되지 않은 것이다.**

## 연결 유형별 가이드

### 상호 링크 (양방향 연결이 필요한 페이지)

| 페이지 A | ↔ | 페이지 B | 연결 방식 |
|----------|---|----------|-----------|
| 로그인 | ↔ | 회원가입 | "계정이 없으신가요?" / "이미 계정이 있으신가요?" |
| 로그인 | → | 비밀번호 찾기 | "비밀번호를 잊으셨나요?" |
| 프로필 | ↔ | 설정 | 탭 또는 상호 링크 |
| 목록 | → | 상세 | 목록 항목 클릭 |
| 상세 | → | 목록 | 뒤로가기 / 브레드크럼 |

### 계층 구조 (부모 → 자식)

```
대시보드 (부모)
├── 프로젝트 목록 (자식)
│   └── 프로젝트 상세 (손자)
│       └── 프로젝트 편집 (증손자)
├── 멤버 관리
└── 설정
```

- 자식 페이지는 **브레드크럼**으로 상위 계층 표시
- 부모 페이지는 자식으로의 **진입 링크** 필수

### 독립 페이지 (법적/정책)

| 페이지 | 진입점 |
|--------|--------|
| 이용약관 | Footer, 회원가입 |
| 개인정보처리방침 | Footer, 회원가입 |
| FAQ | Footer, GNB(선택) |

## 리다이렉트 연결

페이지 간 이동 시 **작업 완료 후 리다이렉트**도 연결에 포함된다:

| 액션 | 리다이렉트 대상 |
|------|----------------|
| 로그인 성공 | 대시보드 (또는 이전 페이지) |
| 회원가입 완료 | 이메일 인증 안내 또는 로그인 |
| 로그아웃 | 홈 또는 로그인 |
| 폼 제출 완료 | 목록 또는 상세 |
| 권한 없는 페이지 접근 | 로그인 (returnUrl 포함) |
| 404 에러 | 홈으로 돌아가기 링크 |

## 금지 사항

1. **라우터에만 등록하고 어디서도 링크하지 않은 페이지 금지**
2. **URL 직접 입력으로만 접근 가능한 페이지 금지** (에러 페이지 제외)
3. **페이지 생성 후 진입점 연결 없이 작업 완료 선언 금지**
4. **네비게이션 맵 미갱신 금지** — 페이지 추가/삭제 시 반드시 맵 업데이트

## 검증 방법

### 수동 검증
1. 브라우저에서 홈(`/`)부터 시작
2. 클릭만으로 모든 페이지에 도달할 수 있는지 확인
3. 도달 불가능한 페이지가 있으면 진입점 추가

### 자동 검증 (선택)
```bash
# Next.js 기준 - 라우트 파일과 링크 비교
# 모든 page.tsx가 최소 하나의 Link/href에서 참조되는지 확인
```
