# 수직 통합 의무화

> 상위: [rules/_index.md](./_index.md)

새 기능을 수평 확장하기 전에, 기존 기능이 실제로 동작하는지 수직 통합을 완료하라.

---

## 규칙

### 1. DB 스키마 변경 시
- 같은 세션 내에 `db:push` 또는 마이그레이션 실행 **필수**
- schema.ts에 컬럼/테이블을 추가하고 마이그레이션을 미루지 않는다

### 2. API 엔드포인트 생성 시
- 같은 세션 내에 최소 1개 프론트엔드 페이지에서 호출을 확인한다
- API만 만들고 UI 연결 없이 다음 기능으로 넘어가지 않는다

### 3. Mock 데이터 기반 UI 작성 시
- 3세션 이내에 실 API/DB 연동 계획을 수립한다
- Mock 기반 페이지가 5개 이상 쌓이면 전환을 우선한다

### 4. 새 소스/어댑터/모듈 추가 시
- 같은 세션 내에 최소 dry-run 또는 단위 검증을 실행한다
- "코드만 작성하고 한 번도 실행하지 않은" 상태로 남기지 않는다

---

## 예외

- 명시적으로 **"프로토타이핑 세션"**으로 선언한 경우
- DB 접속 환경이 미구성된 초기 단계 (Phase 0~1)
- 사용자가 의도적으로 보류를 지시한 경우

---

- Mock→실 API 전환이 8세션 동안 미착수
